# Project Journal – Taniti Prototype

## Monday

Today was basically just trying to figure out what this project even is.

At first it felt like a lot — timeline, persona, wireframes, prototype, testing… just a bunch of stuff thrown at me.

So I slowed down and broke it into pieces.

Main thing was figuring out the pages:
home, dining, lodging, activities, transportation, faq

Once that clicked, it started to feel more doable.

Kept thinking:
“If I was going to this island, what would I actually want to see first?”

That helped me stop overthinking it.

Also started picturing how I want this to feel definitely tropical, clean, nothing cluttered. Just easy to move through.

## Tuesday

Today I set everything up.

Made all the HTML pages, styles.css, script.js, and created folders for images so things don’t get messy later.

Right away messed up a couple things:
forgot to link CSS on some pages
and messed up file paths

Took a minute but got that fixed.

Also spent a lot of time looking for images.

Checked:
Google Images (quick ideas)
Adobe Stock (way better quality)

I didnt want to just grab random stuff I want everything to match:
beach images for home
food for dining
hotel/resort for lodging
real-looking activities

Also started looking through Bootstrap stuff today. Not copying it exactly, just getting ideas for layout and structure.

## Wednesday

Today was kind of messy but productive.

Started putting content into the pages and it just didn’t feel right at first.

Some sections felt off and some pages felt empty.

Moved things around a lot until it started to make more sense.

Kept asking myself:
“does this feel easy to read?”
“does this actually make sense to someone visiting?”

Decided to:
keep nav the same everywhere
break things into sections
stop using big blocks of text

Started using Bootstrap grid more (rows/columns) and that helped a lot with layout.

Also started matching images with sections instead of just dropping them in randomly.

That made a big difference.

## Thursday

Today everything started looking like an actual site.

Built out all the pages using the same structure:
navbar → hero → sections → footer

That consistency helped a lot.

Used Bootstrap for:
navbar
grid layout
spacing
grabbed some section stuff from bootstrap

css was a headache for me today, I used bootstrap css but wanted to use my own CSS to make it look how I want and kept hitting walls. I forgot alot of bootstrap uses same class names, it took awhile but i finally adjusted everything and made it work.

Spent a lot more time fixing images today.

Before I was just placing them and it didn’t look right.

Now I’m paying attention to:
image sizes
spacing
how they line up with text

Ran into the same annoying problems again:
links not working
images not loading

Yep! file paths again.

Fixed them and kept moving.

## Friday

Today was styling.

Spent most of the time in styles.css.

Set up colors:
blues for water
greens for island
sand colors for background

Also started building reusable stuff:
feature cards
hero sections
cta sections

Bootstrap helps with layout, but the look is definitely coming from the CSS.

Worked a lot on:
spacing
alignment
making things feel balanced

Some stuff looked off at first:
spacing uneven
images too big
things not lining up

Just kept adjusting until it felt right.

FAQ page actually came out pretty nice with the background and overlay.

## Saturday

Today was testing and fixing stuff.

Also added the JavaScript for the menu toggle.

At first it didn’t work had to mess with it for a bit.

Ended up being a selector issue.

After that I went through everything:
clicked every link
checked every page
looked at every image

Found:
a few broken links
some images didn’t look right in the layout

Went back and fixed those.

Testing definitely helped more than I thought it would.

## Sunday

Last day.

Went through everything again and cleaned it up.

Fixed:
spacing issues
layout inconsistencies
random little CSS problems

Also checked images again:
made sure they fit
made sure they match the content
got rid of anything that felt off

Then uploaded everything to GitHub and got the site live.

That felt really good.

Had to tweak a couple small things after, but nothing major.

## Final Thoughts

Honestly this turned out pretty solid.

What worked:
keeping things simple
using Bootstrap for structure
using my own CSS combined with bootstrap for design
actually thinking about image placement

What didn’t:
file paths… over and over
some early layout decisions
just throwing images in without thinking

Big thing I learned:
layout and images matter way more than I thought

Bootstrap helps, but you still have to know what you’re doing.

If I did this again:
plan layout better first
be smarter with images from the start
test earlier
double check paths every time

## Done

Everything works:
all pages done
links working
images loading
layout looks clean
site is live

Feels good to be done with this one. Now lets just see if it passes or not lol
