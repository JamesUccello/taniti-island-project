# Taniti Tropical Prototype

Project Goal: build a clean, working multi-page prototype for the Taniti island project and get a live github URL for submission.



## My Game Plan

### 1. Understand what needs to be done
- Look through instructor notes and templates
- Identify needed pages, folders, content, images, bootstrap content/code, and project requirmnets
  - timeline
  - persona
  - wireframe
  - testing
  - prototype
  - usability questions
  - what .html pages do I need
  - What folders do I need
  - research Taniti Island to get a vibe about what I want my site to look like
  - Get needed images from Adobe Stock
  - Find bootstrap sections to use




### 2. Set up the project
- Create project folder
- Create all HTML files:
  - index
  - dining
  - lodging
  - activities
  - transportation
  - faq
- Create styles.css
- Create script.js
- Create images folder with subfolders:
  - home
  - dining
  - lodging
  - activities
  - transportation
  - faq
  - logo



### 3. Use the island content
- Use info from “About the Island”
- Break content into the correct pages



### 4. Plan the layout
- Keep navigation simple and the same on every page
- Think like someone visiting for the first time:
  - food
  - places to stay
  - things to do
  - getting around
  - important info
- Make everything easy to find
- My Footer logo



### 5. Pick a look
- Go with tropical colors:
  - blues (water)
  - sand tones
  - greens
- Keep it clean and readable
- (KISS) Don’t overdo styling



### 6. Add images
- Find images on Adobe Stock and the web that match each page
- Put them in the correct folders
- Match images to the content sections



### 7. Build the pages
- Create each page one at a time
- Add:
  - navigation
  - content
  - images
  - footer with logo
- Make sure all pages link to each other



### 8. Style everything
- Use styles.css for all styling
- Keep layout consistent across pages
- Fix spacing and alignment
- Make sure text is easy to read
- FAQ page (use gradient)



### 9. Add JavaScript 
- Use for small things like menu toggle
- Keep it simple 



### 10. Test everything
- Open all pages in browser
- Click every link
- Check all images load
- Fix anything broken or off



### 11. Publish
- Upload everything to GitHub
- Turn on GitHub Pages
- Get the live URL



## Done when:
- All pages work
- No broken links or images
- Layout looks clean
- URL is live and ready to submit
